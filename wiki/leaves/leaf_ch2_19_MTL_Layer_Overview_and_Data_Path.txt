<h level="2">MTL Layer Overview and Data Path</h>
<info>
  <p>Chapter: [[架构]]</p>
  <p>Source: <code>ch2 p83</code></p>
  <p>Confidence: <code>high</code></p>
</info>

<p><em>MAC Transaction Layer（MTL）</em> 提供 FIFO 内存接口，用于缓冲和调节系统内存与 MAC 之间的数据包。MTL 还负责在应用时钟域与 MAC 时钟域之间传输数据。</p>

<p>MTL 包含两条数据路径：</p>
<ul>
  <li><em>Transmit Path</em>：通过 <em>Application Transmit Interface（ATI）</em> 与应用通信</li>
  <li><em>Receive Path</em>：通过 <em>Application Receive Interface（ARI）</em> 与应用通信</li>
</ul>

<p>MTL 同时提供 <em>MAC Control Interface（MCI）</em> 作为控制路径。数据路径宽度支持 32-bit、64-bit 或 128-bit。</p>

<h level="3">Transmit Path 数据流</h>
<ul>
  <li>应用或内部 DMA 通过 ATI 将 Ethernet 数据包推入对应的 Tx Queue</li>
  <li>当队列达到阈值（Threshold mode）或完整包已存入（Store-and-Forward mode）时，数据被弹出并传输到 MAC</li>
  <li>EOP 传输完成后，MTL 从 MAC 获取发送状态并回传给应用或 DMA</li>
  <li>Tx Queue 默认大小为 2K bytes</li>
  <li>应用或 DMA 通过 ATI 接口的 SOP 和 EOP 信号标识包边界</li>
  <li>多队列配置下，应用或 DMA 需通过 ATI 指示目标队列号</li>
</ul>

<h level="3">Receive Path 数据流</h>
<ul>
  <li>MTL Rx 模块从 MAC 接收数据包并推入 Rx Queue</li>
  <li>队列状态（fill level）在超过接收阈值或完整包接收后通知应用或 DMA</li>
  <li>MTL 指示队列 fill level，使 DMA 能够以预配置的 burst 长度发起 AHB/AXI 传输</li>
</ul>

<h level="3">相关寄存器</h>
<ul>
  <li><kbd>MTL_Operation_Mode</kbd>：MTL 全局操作模式</li>
  <li><kbd>MTL_TxQ0_Operation_Mode</kbd>：Tx Queue 操作模式（含 TTC 阈值、TQS 队列大小）</li>
  <li><kbd>MTL_RxQ0_Operation_Mode</kbd>：Rx Queue 操作模式（含 RTC 阈值、RQS 队列大小）</li>
</ul>
