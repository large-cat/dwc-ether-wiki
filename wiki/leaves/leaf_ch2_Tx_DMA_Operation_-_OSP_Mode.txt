<h level="2">Tx DMA Operation - OSP Mode</h>
<p>OSP（Operate on Second Packet）模式下，对应 DMA channel Transmit Control Register 的 Bit 4 置位后，Transmit 进程可同时获取两个包，而无需先关闭第一个包的状态 descriptor。第一个包传输完成后，DMA 立即轮询 Transmit Descriptor list 获取第二个包；若第二个包有效，则先传输第二个包，再写入第一个包的状态信息。</p>

<p>OSP 模式下 Run 状态的 Tx DMA 工作流程：</p>

<ol>
  <li>DMA 执行 default 模式的步骤 1 至 7（配置 descriptor、更新 Tail pointer、仲裁、获取 descriptor、检查所有权、解码地址、传输数据到 MTL）。</li>
  <li>DMA 获取下一个 descriptor，但不关闭前一个包的最后一个 descriptor。</li>
  <li>若 DMA 拥有该 descriptor，解码 transmit buffer 地址；若不拥有，进入 Suspend 模式并跳至步骤 7。</li>
  <li>DMA 从系统内存获取第二个包的数据并传输到 MTL，直到 EOP 数据传完；若该包跨多个 descriptor，关闭中间 descriptor。</li>
  <li>DMA 等待前一个包的传输状态和时间戳。状态可用后，若捕获了时间戳（由 status bit 指示），将其写入 <kbd>TDES0</kbd> 和 <kbd>TDES1</kbd>；随后将状态（Own bit 清除）写入对应的 <kbd>TDES1</kbd>，关闭该 descriptor。若前一个包未启用时间戳，DMA 不修改 <kbd>TDES2</kbd> 和 <kbd>TDES3</kbd>。</li>
  <li>设置 Transmit 中断（若使能）。DMA 获取下一个 descriptor 并返回步骤 3（状态正常时）。若前一个传输状态显示 underflow 错误，进入 Suspend 模式（步骤 7）。</li>
  <li>Suspend 模式下：
    <ul>
      <li>若收到来自 MTL 的 pending 状态和时间戳，DMA 写入时间戳（若当前包启用）到 <kbd>TDES0</kbd>/<kbd>TDES1</kbd>，写入状态到对应 <kbd>TDES1</kbd>，设置相关中断，然后返回 Suspend 模式。</li>
      <li>若无 pending 状态，且应用通过清除对应 DMA channel Transmit Control Register 的 Bit 0 停止 DMA，则进入 Stop 状态。</li>
    </ul>
  </li>
  <li>DMA 只有在收到对应 channel Transmit Descriptor Tail Pointer register 的 Transmit Poll demand 后，才能退出 Suspend 模式并返回 Run 状态（根据 pending 状态决定返回步骤 1 或 2）。</li>
</ol>

<important>
  <p>由于 DMA 在关闭当前 descriptor 之前就会获取下一个 descriptor，descriptor ring 长度必须大于 2。Synopsys 建议 descriptor 长度至少为 4。</p>
</important>
