<p>AHB Master Interface（EQOS-AHB 配置）操作细节：</p>

<h level="3">一、AHB Memory Read</h>
<ul>
  <li>使用 INCR4 burst 从系统内存读取数据</li>
  <li>不产生 WRAP burst、locked 或 protected 传输（无 <kbd>HLOCK</kbd> / <kbd>HPROT</kbd> 信号）</li>
  <li>DMA 仅在能完全接收 burst 数据时才请求 AHB Burst Read</li>
  <li>读数据无延迟直接推入 DMA（多通道时可能因共享内存产生延迟或插入 Busy 周期）</li>
</ul>

<h level="3">二、AHB Memory Write</h>
<ul>
  <li>使用 INCR4 burst 向系统内存写入 Ethernet 报文</li>
  <li>Fixed burst 模式下，EOP 提前到达时剩余 beats 用 dummy data 填充</li>
  <li>INCR 模式下可提前终止突发</li>
</ul>

<h level="3">三、Early Burst Termination (EBT)</h>
<ul>
  <li>AHB master 收到 EBT 后重新请求总线，用 SINGLE 完成剩余传输</li>
</ul>

<h level="3">四、Error Response 处理</h>
<ul>
  <li>ERROR 响应会停止该 DMA <em>所有后续传输</em></li>
  <li>通过 CSR 和 <kbd>sbd_intr_o</kbd> 中断报告 <em>Fatal Bus Error</em></li>
  <li>应用可选择硬复位或软复位重启模块</li>
</ul>

<h level="3">五、Split/Retry Response</h>
<ul>
  <li>收到 RETRY / SPLIT 后立即重建事务并重新请求总线</li>
  <li><kbd>hreq_o</kbd> 仅在采样响应的第一周期 de-assert 一个时钟</li>
  <li>SPLIT 响应时第二次 <kbd>hreq_o</kbd> 请求会被 AHB arbiter 屏蔽，直到 slave 准备好数据</li>
</ul>

<h level="3">六、Descriptor Write-Back</h>
<ul>
  <li>64-bit AHB 上，大端配置时 32-bit status 写入 <kbd>hwdata_o</kbd> 高 32 位</li>
  <li>小端配置时 32-bit descriptor data 写入低 32 位</li>
</ul>
