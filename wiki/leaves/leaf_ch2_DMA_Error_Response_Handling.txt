<h level="2">DMA Error Response Handling</h>
<p>对于 DMA channel 发起的任何数据传输，若 slave 回复 error response，DMA 会停止所有操作，并更新对应 DMA channel Status Register 中的 error bits 和 Fatal Bus Error bit。应用可以选择对 DWC_ether_qos 执行复位，或重新初始化 DMA descriptor list 后重新启动。其他 DMA channel 不受该错误影响。</p>

<p>不同配置下的 error response 路径：</p>

<ul>
  <li><strong>EQOS-DMA：</strong> 通过 <kbd>mdc_error_i</kbd> 信号接收 error response（见 Figure 2-31）。</li>
  <li><strong>EQOS-AHB：</strong> 通过 <kbd>hresp_i</kbd> 信号接收 error response（见 Figure 2-13）。</li>
  <li><strong>EQOS-AXI：</strong> 通过 <kbd>rresp_m_i</kbd>（见 Figure 2-16）或 <kbd>bresp_m_i</kbd> 信号接收 error response。</li>
</ul>

<p>收到 error response 后，对应的 DMA（Rx 或 Tx）进入 Fatal Error 状态，所有操作停止。应用必须提供 soft reset 以重新启动 DMA 操作。</p>

<warning>
  <p>仅发生错误事务的特定 DMA channel 会受到影响并进入 STOP 状态，其他 channel 的流量不受影响。</p>
</warning>
