<h level="2">系统架构概览</h>

<p>DWC_ether_qos 可通过 coreConsultant 配置为以下架构：</p>

<ul>
  <li><strong>EQOS-AHB</strong>（默认）— DMA + AHB</li>
  <li><strong>EQOS-AXI</strong> — DMA + AXI</li>
  <li><strong>EQOS-DMA</strong> — DMA + native 接口，无 AHB/AXI</li>
  <li><strong>EQOS-MTL</strong> — 仅 FIFO 层 + native FIFO 接口，无 DMA</li>
  <li><strong>EQOS-CORE</strong> — 仅 native FIFO 接口，无 DMA/MTL/AHB/AXI</li>
</ul>

<p>速率配置可选：10/100/1000 Mbps、仅 10/100、或仅 1000 Mbps。</p>

<h level="3">Core 内部结构</h>

<p>Figure 1-1 进一步展示了 Core 内部的子组件：Host DMA Interface（AXI/AHB Master）和 CSR Slave Interface（AHB/AXI/APB）是 Core 对外的两个主接口；DMA Controller 内嵌 DMA Arbiter 和最多 8 组 Tx/Rx Channel；MTL 包含独立的 Tx Queues 和 Rx Queues；MAC Core 包含 Transmit Protocol Engine 和 Receive Protocol Engine；帧处理完成后经 PHY Interface MUX 输出。</p>

<h level="3">辅助模块（Sideband）</h>

<p>Interrupt Controller 汇总 DMA、MTL、MAC 的事件并产生中断；Timestamp Module 与 MAC Core 交互完成 PTP 时钟同步；MDIO/SMA 用于外部 PHY 的管理和状态访问。这些模块通过独立的事件/控制线连接到 Core，不占用主数据通路。</p>
