<h level="2">系统架构概览</h>

<p>DWC_ether_qos 可通过 coreConsultant 配置为以下架构：</p>

<ul>
  <li><strong>EQOS-AHB</strong>（默认）— DMA + AHB</li>
  <li><strong>EQOS-AXI</strong> — DMA + AXI</li>
  <li><strong>EQOS-DMA</strong> — DMA + native 接口，无 AHB/AXI</li>
  <li><strong>EQOS-MTL</strong> — 仅 FIFO 层 + native FIFO 接口，无 DMA</li>
  <li><strong>EQOS-CORE</strong> — 仅 native FIFO 接口，无 DMA/MTL/AHB/AXI</li>
</ul>

<p>速率配置可选：10/100/1000 Mbps、仅 10/100、或仅 1000 Mbps。</p>

<h level="3">系统级框图</h>

<p>Figure 1-1 展示了 DWC_ether_qos 的系统级框图（System-Level Block Diagram）。数据通路自左向右：System Memory 通过 Host DMA Interface（AHB/AXI Master）与 DMA Controller 交互；DMA Arbiter 仲裁所有通道的收发路径；MTL（MAC Transaction Layer）的 Tx/Rx Queues 桥接 DMA 与 MAC Core；MAC 的 Transmit/Receive Protocol Engine 完成帧处理后，经 PHY Interface MUX 发往外部 PHY。CSR Slave Interface 为 Host CPU 提供 Control &amp; Status 访问；Interrupt Controller、Timestamp Module 和 MDIO/SMA 为辅助模块。</p>
